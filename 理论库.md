**首次使用 v8.2 时,在 GitHub 仓库创建以下文件:**

**`理论库.md` 初始内容**:

```markdown
# 投研理论库 · v8.2

> 本文件由每日晨报自动追加。每个理论在被晨报展开后,自动收录于此。
> 同一理论会有多次记录,每次记录都对应一个具体应用情境——
> 这是间隔重复学习的物理实现。

## 索引(按字母排序,带最新更新日期)

(空。每次有新理论展开后,追加到下方"详细记录"区,
并在此处更新索引名+最新出现日期。)

## 详细记录(按时间倒序)

### 模板示例

---

**理论名**:DCF(现金流折现模型)
**类别**:财务类
**首次出现日期**:2026-05-12
**累积出现次数**:1

**首次展开内容**(完整四要素):

【是什么】 ...
【为什么】 ...
【怎么计算】 ...
【历史案例】 ...

**应用情境记录**:
- 2026-05-12:用于评估 XX 公司估值合理性,推演结论是 [...]

---

(后续出现时,在"累积出现次数"+1,在"应用情境记录"中追加一行)
```

## v8.2 与 v8 的差异

| 项目 | v8 | v8.2 |
|------|----|------|
| 推演中遇到理论 | 直接使用 | 强制原地展开(财务/宏观/产业类完整四要素) |
| 同一理论重复出现 | 不做特殊处理 | 仍然完整展开,记录"累积出现次数" |
| GitHub 提交清单 | 3 个必交文件 | 新增第 4 个:`理论库.md` |
| 月度复盘 | 无 | 推荐月底生成"理论库索引+高频理论排名" |
| 晨报长度 | 3000-5000 字 | 6000-10000 字 |
| 阅读时间 | 5-10 分钟 | 15-20 分钟 |

## 月底可选动作:生成"理论库索引+高频理论排名"

每月底,可以让 DeepSeek 跑一次:

```
任务:读取我的 `理论库.md`,生成本月理论复习总结:
1. 本月新增了多少条新理论?具体清单
2. 本月哪些理论出现频次最高?(前 5 名)
3. 高频理论的应用情境呈现什么共性?(是否暴露某种市场结构特征)
4. 哪些理论只在某一天出现过、之后再没出现?
   → 是否说明这些理论可能暂时用不到,可以归入"备用知识"?
5. 我下个月应该重点关注哪类理论?(基于 A9 趋势池演化方向)
```

把结果存为 `理论库月报_YYYY-MM.md`。
